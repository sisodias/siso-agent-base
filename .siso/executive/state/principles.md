# Executive Principles

1. Filesystem first, database later only if needed.
2. Markdown first, structured indexes later only if needed.
3. The executive owns durable state; the mouth owns conversation.
4. Inbox before memory: raw ideas should not instantly become permanent truth.
5. Decisions must be explicit.
6. Tasks must have a next action.
7. Worker agents should be scoped, not vague.
8. Reviews should recommend action, not just summarize.
9. Human approval is required for risky, broad, or irreversible changes.
10. The system should be easy to inspect, correct, and reset.
