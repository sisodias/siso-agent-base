# Workflow: executive-capture

Purpose: capture important conversation items into the executive inbox without instantly polluting permanent state.

## Capture when

- Shaan states a durable preference.
- Shaan makes or confirms a decision.
- A new project/task/research lead appears.
- A blocker is discovered.
- A future follow-up is needed.

## Do not capture when

- It is casual wording with no durable impact.
- It is a temporary frustration unless it reveals a workflow failure.
- It duplicates existing state.

## Inbox event template

```md
# Inbox Event: <short title>

Date: YYYY-MM-DD
Type: idea | request | decision | task | preference | research-lead | blocker | question | source | follow-up
Source: chat | file | worker | research | system
Status: new

## Raw note

## Possible executive handling

- [ ] add task
- [ ] update project
- [ ] record decision
- [ ] add research lead
- [ ] ask clarification
- [ ] archive/discard
```
